## Prerequisites

Before you begin, you'll need:

<Check>An Outlit account with an organization</Check>
<Check>Your public key from Settings → Website Tracking</Check>

## Step 1: Add the Tracker

Choose your preferred integration method:

<Tabs>
  <Tab title="Script Tag">
    Add this script tag before the closing `</body>` tag on every page:

    ```html
    <script
      src="https://cdn.outlit.ai/outlit.js"
      data-public-key="pk_your_public_key_here"
      async
    ></script>
    ```

    <Tip>
      Replace `pk_your_public_key_here` with your actual public key from Settings.
    </Tip>
  </Tab>
  <Tab title="NPM">
    Install the package:

    ```bash
    npm install @outlit/tracker
    # or
    pnpm add @outlit/tracker
    # or
    yarn add @outlit/tracker
    ```

    Initialize once at app startup:

    ```typescript
    import outlit from '@outlit/tracker'

    outlit.init({ publicKey: 'pk_your_public_key_here' })
    ```
  </Tab>
  <Tab title="React">
    Install the package:

    ```bash
    npm install @outlit/tracker
    ```

    Wrap your app with the provider:

    ```tsx
    // app/layout.tsx or _app.tsx
    import { OutlitProvider } from '@outlit/tracker/react'

    export default function RootLayout({ children }) {
      return (
        <OutlitProvider publicKey="pk_your_public_key_here" trackPageviews>
          {children}
        </OutlitProvider>
      )
    }
    ```
  </Tab>
</Tabs>

## Step 2: Verify Installation

After adding the tracker, visit your website and:

1. Open browser DevTools → Network tab
2. Filter by "outlit" or "events"
3. You should see requests to `app.outlit.ai/api/i/v1/...`

<Note>
  Look for requests to `app.outlit.ai/api/i/v1/` with a 200 status code.
</Note>

<Info>
  If you don't see requests, check that your public key is correct and the domain is allowed in your tracking settings.
</Info>

## Step 3: Track Custom Events

Beyond automatic pageviews and form captures, track events that matter to your business:

<CodeGroup>
```typescript Script Tag / NPM
// After a user views pricing
window.outlit.track('pricing_viewed', {
  plan: 'enterprise'
})

// After a user starts a trial
window.outlit.track('trial_started', {
  plan: 'pro',
  source: 'pricing_page'
})
```

```tsx React
import { useOutlit } from '@outlit/tracker/react'

function PricingButton() {
  const { track } = useOutlit()

  return (
    <button onClick={() => track('pricing_clicked', { plan: 'pro' })}>
      Start Free Trial
    </button>
  )
}
```
</CodeGroup>

## Step 4: Identify Visitors

When a visitor signs up, logs in, or submits a form with their email, identify them:

<CodeGroup>
```typescript Script Tag / NPM
// After login or signup
outlit.identify({
  email: 'user@example.com',
  userId: 'usr_abc123', // Your internal user ID (optional)
  traits: {
    name: 'Jane Doe',
    company: 'Acme Inc',
    plan: 'pro'
  }
})
```

```tsx React
import { useOutlit } from '@outlit/tracker/react'

function LoginForm() {
  const { identify } = useOutlit()

  const handleLogin = async (email: string) => {
    // ... your login logic
    
    identify({
      email,
      traits: { lastLogin: new Date().toISOString() }
    })
  }
}
```
</CodeGroup>

<Warning>
  Always call `identify()` after a successful login or signup, not before. This ensures the visitor's anonymous history is properly linked to their profile.
</Warning>

## What's Captured Automatically

With the default configuration, Outlit automatically captures:

| Event | Trigger | Data |
|-------|---------|------|
| **Pageview** | Every navigation | URL, path, title, referrer, UTM params |
| **Form Submit** | Form submission | Form ID, field values (sensitive fields removed) |
| **Session Start** | First visit | Landing page, referrer, UTM attribution |
